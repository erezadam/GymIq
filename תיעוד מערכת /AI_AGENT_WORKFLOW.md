## מטרת המסמך
מסמך זה מרכז את הידע, התובנות והפרקטיקות שנבנו בעבודה מעשית עם סוכן AI לפיתוח תוכנה.
המטרה היא עבודה מהירה אך מבוקרת, עם למידה מצטברת והפיכת הסוכן לשותף אמיתי.

---

## עיקרון יסוד
אם זה לא כתוב – זה לא קיים.  
זיכרון אמיתי הוא מסמכים וקוד, לא שיחה.

## שכבות העבודה
### Agent
הסוכן מבצע. חזק ומהיר, אך תלוי בהקשר שאתה מספק.

### Memory
זיכרון הוא תיעוד:
- CLAUDE.md – חוקי עבודה
- spec.md – החלטות פיתוח
- decision-log.md – למה החלטנו
- קוד + git history

### Hooks
נקודות עצירה מחייבות בזמן התהליך:
- פתיחת סשן
- לפני סיום משימה
- אחרי שינוי משמעותי
- סוף סשן

### Skills
יכולות חוזרות שמוגדרות מראש:
כשמבקשים X → מתבצע Y  
בלי ניסוחים מחדש ובלי ניחושים.

## Hooks מרכזיים
### Session Start Hook
חוסם. אין קוד בלי פתיחה.

### Pre-Completion Hook
לפני סיום:
- בדיקת מובייל
- RTL
- lint
- build במידת הצורך

### Post-Change Learning Hook
האם זו טעות חוזרת או החלטה מערכתית?
אם כן – מציעים עדכון למסמכים.

### Session End Hook
סיכום:
- מה השתנה
- מה למדנו
- מה נשאר פתוח
- האם נדרש עדכון Memory

## Trust Boundaries
### Safe
קריאה, build, lint.

### Guarded
refactor בינוני, שינוי types או store – חובה דיווח.

### Restricted
שינוי config, dependencies, auth, deploy – אסור בלי אישור.
